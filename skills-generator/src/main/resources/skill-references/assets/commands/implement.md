# implement

## Purpose
Execute feature implementation with test-driven development workflow.

## Usage
```
/implement <input-format> [<options>]
```

## Input Formats Supported
- Gherkin feature files (.feature)
- User story documents
- Task descriptions
- OpenSpec proposals

## Workflow
1. Parse requirements from input
2. Generate test scaffolds
3. Implement business logic
4. Run test suite
5. Validate against requirements
6. Report implementation status

## Frameworks
- JUnit 5
- AssertJ
- Mockito
- GherkinJava

## Output
- Generated source code
- Comprehensive test suite
- Implementation report with coverage metrics
- Build verification results
